브랜치는 작업 목적에 맞게 아래 규칙을 따릅니다:

- feature/기능명: 새로운 기능 개발
- fix/버그명: 버그 수정
- refactor/리팩토링대상: 코드 구조 개선
- style/UI작업: CSS/UI 변경
- docs/문서명: 문서 관련 작업
- test/테스트명: 테스트 코드 작업
- chore/환경설정: 설정, 패키지, 빌드

예시:
```
feature/login-form
fix/modal-close-error
```

### 커밋 태그 목록
- Feat: 새로운 기능 추가
- Fix: 버그 수정
- Refactor: 코드 리팩토링
- Style: UI 및 스타일 수정
- Docs: 문서 작성 및 수정
- Test: 테스트 코드 추가/수정
- Chore: 환경 설정, 패키지 등 작업

예시:
```
[Feat] 로그인 기능 구현
[Fix] 펀딩 페이지 정렬 오류 수정
```
